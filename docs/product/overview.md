# Обзор продукта

## Краткое описание

`Lawyer5RP MVP` — веб-приложение для ведения структурированных юридических и околоюридических игровых документов.
На старте проект делает ставку не на широкий каталог форм, а на один главный сценарий: жалоба в ОГП с выводом итогового BBCode для форума.

## Для кого проект

В рамках MVP рассматриваются три основные группы пользователей:

- обычный игрок, который подает обращение от своего имени
- адвокат, который может подавать обращение как от себя, так и в интересах другого лица
- супер-админ, который управляет доступом на уровне аккаунта внутри встроенной админки

## Что должно решить MVP

- собрать в одном месте данные аккаунта, сервера, персонажей, доверителей и документов
- убрать ручное дублирование сведений из персонажа в документ
- снизить количество ошибок в обязательных полях жалобы
- обеспечить повторяемую генерацию итогового форумного BBCode
- сохранить прозрачный жизненный цикл документа через слепки и версии

## Главный продуктовый сценарий

Основной сценарий MVP:

1. Пользователь входит в систему.
2. Пользователь выбирает серверный контекст.
3. Пользователь выбирает активного персонажа.
4. Пользователь запускает мастер жалобы в ОГП.
5. Пользователь заполняет обязательные данные, доказательства и, при необходимости, блок представителя.
6. Система генерирует итоговый BBCode для просмотра и копирования.
7. После публикации пользователь вручную указывает ссылку на публикацию форума.

## Базовые принципы продукта

- Один full-stack проект без отдельного backend-сервиса.
- Админка находится внутри того же приложения.
- Все ключевые связи строятся вокруг модели `Account -> Server -> Characters -> Documents`.
- Документ после первого сохранения черновика начинает жить на основе слепка.
- Ссылки на публикации и доказательства используются вместо загрузки файлов.
- AI используется как вспомогательный серверный инструмент и не является центральной частью сценария.

## Что не является целью текущего MVP

- универсальный конструктор любых документов
- редактируемый BBCode
- загрузка и хранение файлов в document flow
- публичный API
- отдельная интеграционная backend-платформа
