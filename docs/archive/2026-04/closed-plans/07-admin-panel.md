# План 07: Admin Panel

## Цель этапа

Добавить встроенную админку внутри того же приложения без отдельного проекта.

## Что входит

- отдельный защищенный раздел внутри основного приложения
- доступ по аккаунтному признаку `super_admin`
- базовые административные представления для контроля данных и справочников

## Что не входит

- отдельное админ-приложение
- внешняя административная API-платформа
- сложная ролевая матрица для команды

## Основные задачи

1. Выделить админский route space внутри того же приложения.
2. Ограничить доступ по `profiles.is_super_admin`.
3. Подготовить базовые представления для просмотра пользователей, серверов и документов.
4. Подготовить минимальные инструменты поддержки справочников, если они нужны для MVP.

## Критические правила этапа

- админка находится внутри того же приложения
- `super_admin` — уровень аккаунта, а не персонажа
- доступ в админку не должен зависеть от активного персонажа

## Критерии завершения

- супер-админ может войти в админ-раздел
- обычный пользователь не получает туда доступ
- базовый административный контроль MVP доступен
