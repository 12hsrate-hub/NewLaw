# Git Workflow

## Назначение

Этот документ фиксирует базовый git flow проекта для MVP.

## Основные ветки

- `feature/*`
- `staging`
- `main`

## Правила работы

### feature/*

- локальная разработка ведется в `feature/*`
- каждая отдельная задача или логический блок изменений оформляется в своей feature-ветке

### staging

- `staging` используется для промежуточной интеграционной проверки
- merge в `staging` выполняется после локальной проверки feature-ветки
- staging должен оставаться рабочей веткой для предрелизной проверки

### main

- `main` используется для production
- merge в `main` выполняется после проверки изменений в `staging`

## Базовый поток

1. Разработка ведется локально в `feature/*`.
2. После локальной проверки изменения интегрируются в `staging`.
3. После промежуточной проверки изменения интегрируются в `main`.
4. Из `main` выполняется production rollout.

## Hotfix

- `hotfix` используется только при реальной необходимости
- после исправления production-проблемы изменения должны быть обязательно возвращены в `staging`
