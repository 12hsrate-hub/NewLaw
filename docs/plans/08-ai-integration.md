# План 08: AI Integration

## Цель этапа

Добавить единственный AI-сценарий MVP без расширения продукта за пределы согласованного scope.

## Что входит

- серверный вызов `OpenAI API`
- улучшение описания для адвоката
- логирование всех запросов в `ai_requests`

## Что не входит

- клиентские вызовы OpenAI
- генерация всего документа через AI
- дополнительные AI-инструменты и ассистенты

## Основные задачи

1. Подготовить серверный модуль вызова OpenAI.
2. Реализовать сценарий улучшения описания для адвоката.
3. Логировать каждый запрос и ответ в `ai_requests`.
4. Сохранить трассировку ошибки и статуса выполнения.
5. Ограничить доступ к AI-сценарию только подходящим ролям и точкам UI.

## Критические правила этапа

- `OpenAI API` вызывается только с сервера
- все AI-вызовы логируются
- MVP AI-сценарий только один: улучшение описания для адвоката

## Критерии завершения

- адвокат может отправить описание на улучшение
- ответ возвращается в рамках серверного сценария
- журнал `ai_requests` отражает успешные и ошибочные вызовы
