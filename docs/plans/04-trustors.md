# План 04: Trustors

## Цель этапа

Реализовать независимый реестр доверителей внутри связки `user + server`.

## Что входит

- список доверителей
- создание с минимальным набором полей
- последующее дозаполнение данных
- soft delete карточки доверителя
- подготовка к использованию доверителя в документе

## Что не входит

- автоматическое обновление старых документов
- загрузка файлов доверителя

## Основные задачи

1. Реализовать CRUD для `trustors`.
2. Разрешить создание неполной карточки.
3. Подготовить выбор доверителя в документном мастере.
4. Зафиксировать, что удаление карточки не ломает исторические документы.
5. Подготовить механизм явного сохранения данных из документного слепка обратно в карточку.

## Критические правила этапа

- доверители привязаны к `user + server`, а не к `character`
- доверитель внутри документа живет как отдельный слепок
- удаление доверителя не должно менять старые документы

## Критерии завершения

- пользователь может создавать и дополнять доверителей
- пользователь может удалять доверителей через soft delete
- доверитель готов к встраиванию в мастер жалобы
